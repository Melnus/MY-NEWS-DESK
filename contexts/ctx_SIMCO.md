# Sim Companiesにおける卸・販売

# 1. 経緯（なぜ卸率・小売率があるか）

Sim Companiesでは市場価格をそのまま使うと：
- プレイヤー間で価格が揃いすぎる
- 役割分担（生産・卸・小売）が成立しない
だから内部的に
> 「同じ商品でも流通段階ごとに利益を分離する」
ための仕組みとして
- 卸（wholesale）
- 小売（retail）
の“マージン階層”が入ってる。

---

# 2. 基本構造（卸・小売の式）
① 卸価格（基本式）
>卸価格 = 生産コスト × (1 + 卸率)

② 小売価格
>小売価格 = 卸価格 × (1 + 小売率)

展開すると：
>小売価格 = 生産コスト × (1 + 卸率) × (1 + 小売率)

③ まとめ形（重要）

>最終販売価格 = C × (1 + w) × (1 + r)
>C = 生産コスト
>w = 卸率
>r = 小売率

---

# 3. 利益の分解

生産者利益（メーカー）

>利益_生産 = 卸価格 - 生産コスト
>= C × w

卸業者利益

>利益_卸 = 小売価格 - 卸価格
>= C × (1 + w) × r

小売業者利益

>利益_小売 = 最終価格 - 卸価格
>（構造的には卸＋小売マージンを吸収）

---

# 4. 重要なポイント（ここが本質）

このモデルの本質はこれ：
> 「価値ではなく“流通段階ごとの加算構造”で利益が決まる」

つまり
原価ベースの倍率ゲーム
マーケット価格ではなく“構造価格”

---

# 5. 実務的な使い方（ゲーム内最適化）

① 生産側の判断
>w（卸率）が高い商品を優先生産
→ 利益が指数的に伸びる

② 卸側の判断
>(1 + w) × r が最大の組み合わせを探す
→ 実質「2変数最適化」

③ 小売側の判断
>需要 × マージン率 = 実効利益
→ 価格より回転率が重要

---

# 6. 管理会計的に言うと

この構造は完全にこれ：
>販売価格 = 原価 × マークアップ連鎖

さらに分解すると：
卸 = upstream margin
小売 = downstream margin

つまり
> サプライチェーン全体の積構造モデル

---

# 7. 関数：

>P(C, w, r) = C · (1+w) · (1+r)

なので
偏微分すると

>∂P/∂w = C(1+r)
>∂P/∂r = C(1+w)
→ 片方上げるともう片方のレバレッジも上がる


---

まとめ

卸・小売は「段階ごとの乗算マージン構造」

基本式は
> P = C(1+w)(1+r)

利益は各レイヤーで分解できる

本質は「流通構造の最適化問題」
