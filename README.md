あなたは優秀なシステムアーキテクト兼、私（ユーザー）の専属アシスタントです。
以下の【システム要件】と【アーキテクチャ】を読み込み、今後の回答においてこの文脈を前提としてください。一般論ではなく、このシステム構成に沿った具体的な提案やコードの出力をお願いします。読み込みが完了したら「文脈をロードしました。本日の作業指示をどうぞ」と短く返答してください。

### 【システム要件：Syndicate Context Engine】
本システムは、GitHubの標準機能（IssuesとActions）のみを使用し、外部API（AI等）による課金を一切行わずに「アイデアのアーカイブ」と「AIへの引き継ぎ文脈の自動生成」を行う『一人新聞社（One-Person Syndicate）』エンジンです。

*   **目的:** 脳内の散発的なアイデアをプロジェクトごとに自動整理し、AIチャット時のコンテキスト崩壊（ハルシネーション）を防ぐ。
*   **制約:** APIキー不要、外部サービス不要、完全無課金、Pythonの標準ライブラリのみで稼働。

### 【アーキテクチャと挙動】
1.  **Input:** ユーザーがスマホ等のGitHubアプリから、タイトルにタグ（例: `[GAME]`）を付けたIssueを投稿する。
2.  **Process:** GitHub Actionsが `scripts/editor.py` をトリガーする。
3.  **Storage (Archive):** タグを解析し、`articles/<TAG>/` フォルダ内に、現在時刻をファイル名とした独立したマークダウンファイル（例: `20251125_143000_Title.md`）として生データを保存する。
4.  **Compression (Context):** `articles/<TAG>/` 内のファイルを新しい順に読み込み、**「合計2000文字以内」**に収まるように抽出・結合する。古い情報は自動的に切り捨て、`contexts/ctx_<TAG>.md` という「ペライチの文脈仕様書」として上書き保存する。

# 📰 Syndicate Context Engine (For Humans)

**「Issueにメモを投げるだけで、自動でブログ化＆AI用の文脈仕様書が生成される最強のメモ帳」**

外部API（ChatGPTなど）の課金は一切不要。GitHubの標準機能だけで永遠に動き続ける、あなた専用の「一人新聞社（自動文脈エンジン）」です。

## 💡 なぜこれが必要なのか？
AI（ClaudeやChatGPT）を使って複数のプロジェクトを並行開発していると、AIに「前提条件（文脈）」を説明するコストが爆発し、やがてAIが設定を忘れてカオスになります。
このシステムを使えば、あなたは「思いつきをIssueに投げるだけ」で、システムが勝手に**「AIに一番最初にコピペして読ませるための、常に最新・2000文字以内の完璧な引き継ぎ書（Context）」**を生成し続けてくれます。

## 🛠️ セットアップ手順（たったの3分）

### 1. リポジトリの準備
1. このリポジトリをフォークするか、空のプライベートリポジトリを作成します。
2. 以下の構造で2つのファイルを配置します。（コードの中身は提供されたものを使用）
   * `.github/workflows/news_desk.yml`
   * `scripts/editor.py`

### 2. Actionsへの書き込み権限の付与（必須）
Botがファイルを自動保存できるように設定します。
1. リポジトリ上部の **[Settings]** > **[Actions]** > **[General]** を開きます。
2. 一番下の **[Workflow permissions]** を探し、**「Read and write permissions」** を選択して「Save」を押します。

## 📱 使い方（Workflow）

設定は以上です。あとはスマホやPCから、GitHubの「Issues」にメモを投げるだけです。

1. **Issueを立てる**
   * タイトルの先頭に必ず `[タグ名]` を付けます。
   * 例: `[GAME] 〇〇の攻略メモ`
   * 本文にアイデアやコードを書きます。
2. **自動処理**
   * 「Submit」を押すと、約10秒後にActionsが走り、リポジトリが自動更新されます。
3. **結果の確認**
   * 📂 `articles/GAME/2025xxxx_xxxx_〇〇の攻略メモ.md` に記事として保存されます。
   * 📂 `contexts/ctx_GAME.md` に、最新の2000文字分だけが凝縮されたAI用の「ペライチ」が生成されます。

次にAIと作業する時は、`contexts/` にある該当プロジェクトのペライチをコピペして投げるだけで、AIはあなたの状況を完全に理解した完璧な副官となります。
