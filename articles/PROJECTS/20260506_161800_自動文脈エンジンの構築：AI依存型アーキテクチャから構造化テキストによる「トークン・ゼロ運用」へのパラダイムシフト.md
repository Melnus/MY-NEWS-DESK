# TITLE: 自動文脈エンジンの構築：AI依存型アーキテクチャから構造化テキストによる「トークン・ゼロ運用」へのパラダイムシフト

# ABSTRACT 
海外の先進的なAI自動整理システムに着想を得て、複数プロジェクトを管理する文脈エンジンの構築を企図した。当初はAIを用いて入力データを自動整理させる構成を検討したが、過剰なトークン消費（浪費）という課題に直面。システム要件を「Inbox / Processed / Handoff」の3層に分解し、人間によるフォーマット構造化（TADC）とスクリプト抽出を採用した結果、AIの自然言語処理を完全に排除。計算コスト・ゼロで最新のコンテキストを自動維持し、属人性を排除して純粋な論理のみをAIに引き継ぐ「一人新聞社」システムを完成させた。

# DETAIL
### 1. 初期構想と「トークン浪費」の課題
*   **動機:** YouTube等で紹介されていた、Obsidian等の外部ツールを活用してAIにインプットとアウトプットを自動整理させるナレッジマネジメントシステムに触発され、同様の環境構築を志向した。
*   **課題:** 生の長文ログやコードをすべてAIに読み込ませて文脈を再構築（Processed化）させるアプローチは、API呼び出しの複雑化と、日常的なメモのたびに莫大なトークンを浪費する非効率な構造であることが判明した。

### 2. 「3ステージ・モデル」への分解と脱AI化の発見
*   **構造の特定:** 文脈管理の本質が「Inbox（生のメモ）」「Processed（知識庫）」「Handoff（AIへの引き継ぎ書）」の3ステージに切り分けられることを特定した。
*   **最適解の導出:** 「AIにやらせようとしていた文脈整理」は、実は人間のコピペや単純な運用ルールで済むレベルまで軽量化できることが判明。「AIの使用」そのものをシステムから削ぎ落とす判断に至った。

### 3. TADCフォーマットの実装と「哲学的な知見」
*   **実装:** AIによる要約を廃止し、人間が入力時に `# ABSTRACT` と `# CONCLUSION` という目印（TADCフォーマット）を付与する。これをPythonの正規表現で抽出・結合し、2000文字のHandoff（ペライチ仕様書）として自動出力するパイプラインを構築した。
*   **知見（属人性の排除とRAGとの比較）:** 過去の全てのログをベクトル検索（RAG）でAIに探させるのではなく、常に「最新の結論」だけを上書き抽出する本手法は、AIに対し「過去の経緯や誰が言ったか」というノイズを排除し、「現在の純粋な論理的状態」のみを強制的に認識させる点で、極めて合理的であることが確認された。

# CONCLUSION
1.  **AIの最適利用の再定義:** 情報の整理や要約といったタスクにAI（トークン）を浪費するのではなく、「情報のフォーマット（構造）」を人間とシステムで規定することで、AIを純粋な「高度推論を行う副官」としてのみ活用するリソース配分の最適解が証明された。
2.  **引き算のエンジニアリング:** 複雑な自動整理システムを模倣・縮小していく過程で、「AIを使わないこと」が最もエラーがなく高速で、コンテキストの純度を高く保てるというパラドックスに行き着いた。
3.  **持続可能なインフラの完成:** 外部APIへの依存と課金リスクを完全に排除し、GitHubの標準機能（IssuesとActions）のみで永続的に稼働する「一人新聞社（Syndicate Context Engine）」が完成した。
